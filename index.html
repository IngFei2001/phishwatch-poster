Abstract

PhishWatch is a lightweight, real-time phishing detection system implemented as a Chrome Manifest V3 browser extension backed by a microservice architecture. This poster focuses on the cloud-backed greylist database, secure API architecture, warning page UI design, and AI chatbot components. A Neon PostgreSQL serverless database stores previously identified phishing URLs for instant blacklist lookups without external API latency. A Go/Gin API backend deployed on Render.com mediates all database access with API key authentication, rate limiting (60 req/min token bucket), and parameterised queries to prevent SQL injection. A full-screen warning page intercepts suspicious sites with structured evidence display, featuring a dark/light theme toggle and 20-second countdown to prevent impulsive bypass. An AI chatbot powered by the Groq API (llama-3.1-8b-instant) is integrated directly into the warning page, providing users with real-time, context-aware security explanations. GitHub Actions workflows ping both services every 5 minutes to prevent free-tier hibernation.

P2 → Users Cannot Identify Phishing

Blacklist tools take days to update while phishing sites go live within hours of domain registration (Vevera et al., 2025)

Contribution & Innovation
  • Cloud-backed blacklist architecture — updates propagate instantly without reinstallation, advancing over static file-based approaches
  • API key auth + rate limiting (token bucket, 60 req/min) — production-grade security on free-tier infrastructure
  • Parameterised queries throughout — SQL injection prevention at database layer
  • Warning page AI chatbot (Groq llama-3.1-8b-instant) — transforms passive block page into interactive security education tool
  • GitHub Actions keep-alive — eliminates cold-start hibernation without paid hosting upgrade
  • Manifest V3 compliant — meets Chrome Web Store security standards, reduces extension attack surface
  • Dark/light theme toggle + 20-second countdown — prevents impulsive bypass, encourages evidence engagement
  • NexaBridge test site (Netlify) — controlled end-to-end detection validation
